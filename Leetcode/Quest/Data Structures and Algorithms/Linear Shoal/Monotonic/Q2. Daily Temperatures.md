## ğŸ§  Solution Approach

### Algorithm: Monotonic Decreasing Stack
The solution uses a **monotonic decreasing stack** to efficiently find the next warmer temperature for each day.

### Key Insights:
1. We maintain a stack that stores indices of temperatures in decreasing order
2. When we encounter a temperature higher than the temperature at the index on top of the stack:
   - We've found a warmer day for that index
   - Calculate the difference in days: `current_index - stack_top_index`
   - Update the answer array
   - Pop from the stack and continue checking

### Time & Space Complexity
- **Time Complexity:** O(n) - Each index is pushed and popped from the stack at most once
- **Space Complexity:** O(n) - For the stack and answer array in the worst case

## ğŸ“ Code Implementation

```java
import java.util.Stack;

class Solution {
    public int[] dailyTemperatures(int[] temperatures) {
        Stack<Integer> stackIdx = new Stack<>();
        int[] answer = new int[temperatures.length];
        
        for (int i = 0; i < temperatures.length; i++) {
            // Check if current temperature is warmer than the temperature
            // at the index stored on top of the stack
            while (!stackIdx.isEmpty() && temperatures[i] > temperatures[stackIdx.peek()]) {
                int prevIdx = stackIdx.peek();
                answer[prevIdx] = i - prevIdx; // Calculate days difference
                stackIdx.pop();
            }
            stackIdx.push(i);
        }
        return answer;
    }
}
```

## ğŸ’¡ Alternative Approaches

### Brute Force (Not Recommended)
- For each day, scan all subsequent days until finding a warmer temperature
- **Time Complexity:** O(nÂ²) - Too slow for constraints (n â‰¤ 10âµ)
- **Space Complexity:** O(1) (excluding output array)

### Why Stack Solution is Optimal
- The monotonic stack approach optimizes by remembering only the temperatures for which we haven't found a warmer day yet
- Each temperature is compared only with relevant previous temperatures, avoiding unnecessary comparisons
